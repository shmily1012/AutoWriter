# 模型适配层（Model Adaptations）

> 借鉴 Anthropic "Scaling Managed Agents" 的核心原则：
> **Harness 中硬编码的假设会随模型进步而过时。**
> 本文件集中管理所有针对当前模型行为的 workaround，与 Agent 的稳定接口分离。
> 模型升级时，只需审查和更新此文件，无需修改 Agent 定义的核心逻辑。

## 适配项清单

### A1: AI 写作风格倾向（影响：文体家）

**症状**: 模型生成的中文叙事文本有明显的"机器味"特征。
**适配策略**: 在文体家 prompt 中注入去AI味约束。

具体规则（编入文体家和编辑的 prompt 中）：

1. **词汇黑名单**：禁用"首先/其次/此外/值得注意的是/总而言之"等过渡词
2. **万能形容词限制**："微微/缓缓/淡淡"每章合计 ≤ 2次
3. **惊叹词限制**："不禁/竟然/居然"每章合计 ≤ 3次
4. **破折号（——）绝对禁止**
5. **句式均匀性**：禁止连续3个以上等长句
6. **排比三连禁止**：禁止"X了A，X了B，X了C"式排比
7. **心理外化要求**：内心活动必须外化为动作/感官

**预期演进**: 随着模型写作能力提升，部分规则可能放宽或移除。定期用 lint 脚本验证是否还需要这些约束。

### A2: 过度服从倾向（影响：编辑、读者）

**症状**: 模型作为评估者时倾向于给出偏高评分，避免"伤害"前序 Agent 的产出。
**适配策略**:
- 编辑和读者作为独立 subagent 运行（隔离 context）
- 在评估 prompt 中明确要求"打分要真实，不要客气"
- 读者 prompt 中不传入蓝图和编剧讨论（防止锚定）

**预期演进**: 如果未来模型在评估独立性上改善，可以考虑放松隔离要求。

### A3: 一次做太多（影响：所有 Agent）

**症状**: 模型倾向于在单次执行中尝试完成过多任务，导致质量下降。
**适配策略**:
- 章节流水线严格按 Step 执行，每个 Step 只做一件事
- 明确约束"一次只做一个 feature/一个场景"
- pipeline-state.md 强制记录每步进度

**预期演进**: 更强的模型可能能安全地合并某些步骤。

### A4: Context Anxiety（影响：长 session）

**症状**: 在早期模型（如 Sonnet 4.5）中，随着 context window 接近上限，模型行为退化——产出质量下降、开始重复、或试图急于完成。
**当前状态**: Opus 4.5+ 已大幅缓解此问题，Opus 4.6 基本不受影响。
**适配策略**:
- ~~强制 context reset 在每个 Step 之间~~ → 已移除
- 保留 pipeline-state.md 持久化机制作为安全网
- Initializer/Coding 分离减轻单次 session 的 context 压力

**预期演进**: 此适配可能完全移除。保持观察。

### A5: 自我评估不可靠（影响：文体家自检）

**症状**: 模型无法可靠地评估自己的产出质量——Generator 对自己的 bug "视而不见"。
**适配策略**: GAN 启发的 Generator/Evaluator 分离：
- 文体家（Generator）不做自我评估
- 编辑 + 读者（Evaluator）独立运行
- lint 脚本提供客观的自动化检查层

**预期演进**: 这是一个较为根本的限制，短期内不太可能消失。保持分离架构。

## 审查流程

### 模型升级时的适配审查清单

当切换到新模型版本时，执行以下审查：

1. **对每个适配项**:
   - [ ] 用新模型运行一章完整流水线
   - [ ] 检查该症状是否仍然存在
   - [ ] 如果已消失 → 标记为"可移除"，但先保留一个版本观察
   - [ ] 如果仍存在但减轻 → 放宽约束参数
   - [ ] 如果仍严重 → 保持不变

2. **检查新模型的新症状**:
   - [ ] 是否出现了之前没有的新行为模式？
   - [ ] 是否需要新增适配项？

3. **更新元数据**:
   - [ ] 更新"当前模型"和"上次审查"日期
   - [ ] 记录审查结论

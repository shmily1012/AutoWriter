## 研究员调用时机

研究员不在主流水线中，而是按需调用：

```
任何Agent执行中 ──需要真实信息──→ 暂停 → 调用研究员 → 获得结果 → 继续执行
作者brainstorm中 ──"查一下XXX"──→ 调用研究员 → 展示结果 → 继续讨论
独立调用 ──/write-novel research──→ 研究员 → 输出备忘录
```

## Brainstorm 交互细节

每个创作类 Agent（架构师/编剧/文体家）的 brainstorm 分三轮：

### 第一轮：初稿展示
Agent 产出初步方案，编排器展示给作者时使用以下格式：

```
──── 架构师初稿 ────

[初稿内容摘要]

────────────────────

💬 请审阅以上内容。你可以：
• 提出修改意见
• 补充想法
• 说"继续"直接采用
• 说"查一下XXX"调用研究员
```

### 第二轮：讨论修改
作者提出反馈后，Agent 根据反馈调整。可以多轮讨论，直到作者满意。

### 第三轮：定稿
Agent 产出最终版本并写入文件。编排器确认文件已保存。

## 章节定稿收尾清单

当一章通过审阅（追读分≥8或作者手动确认通过）后，执行以下步骤：

1. ☐ 将 `drafts/chapter-{N}-draft.md` 复制到 `chapters/chapter-{NNN}.md`
2. ☐ 架构师更新 `story-state.md`：
   - 添加本章事件到时间线
   - 更新人物位置/状态
   - 更新伏笔状态
   - 记录章末钩子
   - 更新章节计数和总字数
3. ☐ 架构师更新 `characters.md`：
   - 新人物添加条目
   - 关系变化更新
   - 心理状态更新
4. ☐ 架构师更新 `outline.md`：
   - 标记章节为"定稿"
   - 记录偏差（如有）
   - 记录新灵感（如有）
5. ☐ 向作者报告更新内容

## 多本书管理

每本书在 `books/` 下有完全独立的工作空间：

```
books/
├── 都市预知者/          # 书名就是目录名
│   ├── story-state.md
│   ├── characters.md
│   ├── outline.md
│   ├── chapters/
│   └── drafts/
└── 金融猎手/
    ├── story-state.md
    ├── characters.md
    ├── outline.md
    ├── chapters/
    └── drafts/
```

切换书时，只需在命令中指定不同书名。状态文件完全隔离，不会串。
